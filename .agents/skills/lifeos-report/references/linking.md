# 链接与引用规范

## 顶头链接

所有规划文件必须在标题和 tags 之后、正文之前放置顶头链接：

### 上层 → 下层（导航）
```markdown
> 下层： [2026-06 月报](../4-monthly/2026-06.md) ｜ [2026-07 月报](../4-monthly/2026-07.md)
```

### 下层 → 上层（返回）
```markdown
> 上层： [2026 Q2 季报](../3-quarterly/2026-Q2.md) ｜ [2026 年报](../2-annual/2026.md)
```

### 日报连续（前后翻阅）
```markdown
> 同层连续： [前一天日报](../6-daily/2026-05-28.md) ｜ [后一天日报](../6-daily/2026-05-30.md)
```

### 周报汇总（本周所有日报）
```markdown
> 本周日报： [05.25](../6-daily/2026-05-25.md) ｜ [05.26](...) ｜ ... ｜ [05.31](...)
```

## 附录引用

- 主文件在合适的地方引用 appendix 链接
- 根据讲解需要，把 appendix 中的关键内容**直接嵌入主文件**
- 嵌入的内容用引用块包裹，标注来源
- 原则：主文件要能独立阅读和理解，appendix 是补充

## 文件命名

| 文件 | 命名格式 | 示例 |
|------|---------|------|
| 日报 | `<YYYY-MM-DD>.md` | `2026-05-29.md` |
| 周报 | `<YYYY>-<MM>-W<WW>.md` | `2026-05-W22.md` |
| 月报 | `<YYYY>-<MM>.md` | `2026-06.md` |
| 季报 | `<YYYY>-Q<N>.md` | `2026-Q2.md` |
| 年报 | `<YYYY>.md` | `2026.md` |
| 附录 | 短横线英文 | `appendix/dingtalk-reputation.md` |
