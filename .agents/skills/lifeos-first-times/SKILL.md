# lifeOS First Times Skill

自动捕获用户的「第一次」体验，记录到 lifeOS 的初体验页面。

## 触发条件

**自动激活**：当用户在对话中提到以下模式时，本 skill 自动生效：

- "第一次 X" / "初次 X" / "首次 X"
- "今天第一次喝了/吃了/去了/试了/用了 X"
- "这是我第一次 X"
- "first time doing X"
- 任何明确表达「此前从未做过/经历/尝试，今天或近期第一次」的语句

**不触发**：
- 用户只是在说"第一次"这个词但不涉及个人体验（如"他第一次世界大战"）
- 用户明确在讨论别人的第一次
- 用户说"不是第一次"

## 数据源（双写）

### 1. Markdown 源文件

**文件**: `apps/web/content/appendix/life/first-times.md`

格式为 Markdown 表格，按年份分区：

```markdown
## 2026

| 日期 | 第一次 | 备注 |
|------|--------|------|
| 07-07 | 敷面膜 | 决定开始认真护肤 |
| 07-08 | 喝蛋白粉 | 第一次喝，感觉像没味道的牛奶 |
```

### 2. JSON 数据文件

**文件**: `apps/web/public/first-times.json`

前端直接 fetch 此文件渲染页面：

```json
{
  "firstTimes": [
    {
      "date": "2026-07-07",
      "title": "敷面膜",
      "note": ""
    },
    {
      "date": "2026-07-08",
      "title": "喝蛋白粉",
      "note": "感觉像没味道的牛奶"
    }
  ]
}
```

## 操作流程

当检测到触发条件时，执行以下步骤：

### Step 1：确认是「第一次」

从用户语句中判断是否为真实的第一次体验。如果不确定，可以自然地问一句确认（"这是你第一次喝蛋白粉吗？"），但不要过度打断对话。

### Step 2：提取信息

| 字段 | 说明 | 示例 |
|------|------|------|
| date | 日期，YYYY-MM-DD。如果用户说"今天"，用当前日期 | 2026-07-08 |
| title | 简洁的标题，2-8 个字。提取核心动作 + 对象 | 喝蛋白粉 |
| note | 备注，用户的感受或背景。可选，留空则填 "" | 感觉像没味道的牛奶 |

### Step 3：双写

#### 3a. 更新 Markdown 源文件

读取 `apps/web/content/appendix/life/first-times.md`，在对应年份的表格中追加一行。如果该年份的表格不存在，创建新的 `## YYYY` 分区和表格头。

新行追加在表格末尾（最新日期在底部）：

```markdown
| MM-DD | title | note |
```

#### 3b. 更新 JSON 数据文件

读取 `apps/web/public/first-times.json`，在 `firstTimes` 数组末尾追加：

```json
{
  "date": "YYYY-MM-DD",
  "title": "title",
  "note": "note"
}
```

```bash
cd /Users/zhengxinyu/lifeOS && pnpm --filter web build
```

### Step 5：自然回应

在对话中自然提及，不生硬地报告"已记录初体验"。例如：

- "第一次喝蛋白粉，感觉像没味道的牛奶——这已经记到你的初体验里了"
- 或者更自然地融入对话，如果对话在继续讨论这个话题，可以在结束后静默记录

## 注意事项

1. **不要重复添加**：写入前检查 JSON 和 Markdown 中是否已存在相同的 title + date 组合
2. **保持简洁**：title 不超过 8 个字，note 不超过 30 个字。长描述截断或让用户确认
3. **Markdown 和 JSON 必须同步**：两份数据源要保持一致。Markdown 是人类可读的备份，JSON 是前端数据源
4. **日期格式**：Markdown 表格中用 `MM-DD`（如 07-08），JSON 中用 `YYYY-MM-DD`（如 2026-07-08）
5. **排序**：Markdown 表格内按日期升序排列（旧在上，新在下）。JSON 数组也按日期升序
6. **note 留空时**：Markdown 中留空（`| 07-08 | 喝蛋白粉 | |`），JSON 中填空字符串（`"note": ""`）
