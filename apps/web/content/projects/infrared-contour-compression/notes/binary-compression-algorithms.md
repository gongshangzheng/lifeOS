# 传统二值压缩算法与其在图像压缩中的应用

> 2026-07-10 整理，系统梳理二值图像压缩的经典方法，为本项目方案选型提供参考

## 概述

二值图像（每个像素 0 或 1）的压缩与灰度/彩色图像压缩有本质不同：信息量更低、统计特性更特殊、可利用拓扑结构。传统二值压缩算法主要围绕**游程编码**、**二维预测**、**算术编码**三条技术线发展。

## 一、核心算法

### 1. 游程编码（RLE, Run-Length Encoding）

最基础的二值图压缩方法，将连续同值像素合并为 `(值, 长度)` 对。

- **原理**：`00000111110000011111` → `(0,5)(1,5)(0,5)(1,5)`
- **优势**：实现极简，无计算开销，对大面积连通区域效果好
- **劣势**：对噪声/交替频繁的图像可能不压缩反而膨胀
- **典型应用**：传真机早期标准、BMP/PCX 格式、二值 mask 存储

### 2. CCITT Group 3 / Group 4（T.4 / T.6）

传真机时代的标准，专为二值文档图像设计。

| 标准 | 核心方法 | 特点 |
|------|---------|------|
| **Group 3 (G3)** | 一维 RLE + Huffman（MH 编码） | 逐行编码，每行独立 |
| **Group 3 MR** | 二维预测（Modified READ） | 当前行以上一行为参考做相对编码 |
| **Group 4 (G4)** | 纯二维预测（MMR） | 去掉行同步码，压缩率远高于 G3 |

- **G3 Modified Huffman (MH)**：对游程长度做 Huffman 编码，黑白分别建表
- **G4 MMR**：当前行以上一行为参考预测，仅编码差异部分
- **典型压缩率**：G3 约 10:1~20:1，G4 可达 20:1~40:1（文本文档）

### 3. JBIG（T.82）与 JBIG2（T.88）

JBIG 是专门为二值图像设计的高效压缩标准，是二值压缩的**标杆方案**。

#### JBIG (1993)

- **核心**：自适应算术编码（QM-coder）+ 上下文建模
- **与 G4 的区别**：用算术编码替代 Huffman，用自适应上下文模型替代固定二维预测
- **压缩率**：比 G4 高约 10%~30%
- **支持**：渐进传输（低分辨率 → 高分辨率逐步细化）

#### JBIG2 (2000)

JBIG 的重大升级，引入了**基于内容自适应**的编码策略：

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| **全文编码** | 文字符号识别 → 字典化 + 位置编码 | 文档/文字图像 |
| **区域编码** | 图像分为文字/半色调/通用区域分别处理 | 混合内容 |
| **通用编码** | 自适应算术编码（类似 JBIG） | 非文字区域 |

- **核心创新**：模板匹配 + 字典复用——相似符号只编码一次，后续用引用
- **压缩率**：对文本图像可达 **3~5 倍于 JBIG**，甚至 10 倍以上
- **有损/无损**：支持无损和有损两种模式（有损模式允许相似字形替换）

### 4. 熵编码方法（底层工具）

| 方法 | 原理 | 二值图场景 |
|------|------|-----------|
| **Huffman** | 变长前缀码，高频符号短码 | 游程长度编码后做熵编码（G3 MH） |
| **算术编码** | 整个序列映射到 [0,1) 区间的一个数 | JBIG/JBIG2 的底层引擎，效率高于 Huffman |

算术编码对二值图的优势：能精确逼近信息熵，不受"每个符号至少 1 bit"的限制。

## 二、在图像压缩中的应用场景

### 1. 文档/传真压缩（最成熟的应用）

二值压缩算法的**起源领域**。扫描文档、合同、表格等本质上就是二值图像。

```
扫描文档 → 二值化 → G4 / JBIG2 → 存储/传输
```

### 2. 二值 Mask / Alpha 通道压缩

在图像/视频编码中，分割 mask、alpha 通道、ROI 标记等本质上是二值图：

- **JPEG 2000** 内部使用 JBIG 风格编码处理 mask
- **视频编码**（H.264/H.265）中 alpha plane 可使用二值编码
- **图像分割**：语义分割的 per-class mask 可用 RLE/JBIG 压缩后存储

### 3. 轮廓/边缘图像压缩（本项目场景）

```
红外图像 → 边缘提取 → 二值轮廓图 → 压缩 → 极低码率传输
```

二值轮廓图的特点：
- **像素稀疏**：轮廓像素占比极低（通常 <5%）
- **拓扑连续**：轮廓是连续曲线，非随机噪声
- **连通性强**：同一轮廓形成长游程

**各算法对轮廓图的适配性排序**：

| 算法 | 适配性 | 原因 |
|------|--------|------|
| **JBIG2 通用模式** | 最优 | 算术编码 + 高阶上下文模型，能利用轮廓的方向性 |
| **JBIG** | 优秀 | 自适应算术编码，比 G4 更高效 |
| **G4 (MMR)** | 良好 | 二维预测利用行间相关性，实现简单 |
| **RLE + Huffman** | 一般 | 对长游程有效，但不利用二维信息 |
| **纯 RLE** | 较差 | 不利用行间冗余，压缩率有限 |

## 三、超低码率极限分析（0.01bpp 约束）

本项目目标：**I 帧 < 0.01 bpp**（CIF 352×288 = 101,376 px → 总预算约 1014 bit）

| 方法 | 预估 bpp | 是否达标 | 说明 |
|------|---------|---------|------|
| JBIG2 通用模式 | 0.02~0.05 bpp | 接近但可能不够 | 像素级压缩的极限 |
| JBIG2 + 轮廓矢量化 | 0.005~0.01 bpp | 可达标 | 结合像素与几何表示 |
| 链码（Freeman/差分链码） | 0.003~0.008 bpp | 可达标 | 跳出像素域，编码轮廓几何 |
| 轮廓参数化拟合 | < 0.005 bpp | 可达标 | 用样条/多边形近似轮廓 |

> **关键判断**：传统二值压缩（JBIG2）在 0.01 bpp 级别接近极限。更极端的码率需要**跳出像素域**，转向**轮廓的几何表示**（链码、样条拟合、多边形近似等），这是从"压缩像素"到"压缩形状"的范式转变。

## 四、对本项目的方案建议

1. **以 JBIG2 通用模式为 baseline**：作为像素级压缩的参考上限
2. **同步探索链码方案**：参考 ECC（Context Adaptive Extended Chain Coding），适合轮廓跟踪编码
3. **做对比实验**：JBIG2 vs G4 vs 链码，在相同轮廓图上测 bpp，量化各方法的极限
4. **如果 0.01 bpp 不够用**：直接上链码/矢量化路线，二值像素级压缩在极端码率下空间有限
